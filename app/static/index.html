Music library
—

Run step 1 first — everything below reads the index it builds. Then work down: each step tells you if it found anything, and you can fix it or move on. Nothing changes your files until you hit Apply in a step, and every change is reversible from Activity.

1
Audit library
A read-only scan that indexes every file's tags, genres, and sizes. Do this first, and again after adding music — the checks below all read what it builds.
2
Complete genres
Give every artist one consistent genre. Proposes one per artist from the curated map, then the majority of existing tags, then optionally MusicBrainz/Claude. Review and edit before applying; a sole Holiday tag is preserved.
Run the Audit (step 1) first.
3
Map unmapped genres
Raw genre strings in your files that match nothing in the controlled vocabulary, so they get dropped. Assign each to a controlled genre (or mark it junk) to teach the vocabulary — then re-Clean on Optimize and future runs normalize them automatically.
Run the Audit (step 1) first.
4
Album genre outliers
Single-artist albums where most tracks share a genre but a few differ or are untagged — usually per-track slips. Sets the odd tracks to the album's genre (the rest stay as-is). Compilations are skipped, since they're legitimately multi-genre.
Run the Audit (step 1) first.
5
Genre cross-check · MusicBrainz / Claude · optional
Catches consistently-wrong genres — an artist whose every track agrees on a genre that an external source contradicts (majority-vote can never fix this). Slow, and genre is subjective, so review before applying. Bounded per run (~150 artists) but remembers what it checked — run it repeatedly to cover the library; disagreements accumulate below.
Run the Audit (step 1) first.
6
Mislabeled album artists · a different fix from genres
Albums whose album-artist matches no track artist — usually a record label or wrong name (e.g. Dido albums filed under the label “Disky”). This is what makes step 2 fragment an artist. Review, uncheck false positives (producer/DJ & classical), then apply — reversible.
Run the Audit (step 1) first.
Music library
—
Process new additions
Runs your configured steps in order over new/changed files. Every engine is resumable, so it only touches what's new.
1
Clean tags
Normalize genres & fill album artists.
Maps genres to a controlled vocabulary and fills missing album artists, in place. One-click reversible from the Activity page. Needs an Audit first.
2
Analyze BPM & energy
Enables tempo/energy playlist rules.
Detects each track's tempo and a 0–100 energy score. Slow (~1–2s/track) but resumable — safe to run overnight.
3
Convert → iPod
Build the AAC mirror for iPod/iTunes.
Transcodes FLAC from your music library to a 256k AAC mirror; MP3/AAC copied as-is, DRM skipped. The library is never modified, and re-runs only convert what's missing or changed.
Apple Music
Music Monster runs on your server and Apple Music runs on your Mac, so it can't sync for you. Run this on the Mac instead — it adds new mirror files, then re-reads tags for tracks already in your library. Nothing is deleted, so play counts, ratings, and playlists survive. Paste it into Script Editor and press Run; it takes a few minutes on a big library and reports what it did. Tracks whose files have gone missing are skipped rather than failing the run.
